# cmux-second-opinion リファレンスガイド

## 概要

cmux-second-opinion は、cmux を使って別の AI エージェントにコードまたは仕様書の独立したレビューを依頼します。レビュアーは別のワークスペースで動作し、元の作成者やプライマリレビュアーの偏見なしにフレッシュな視点を提供します。

## フロー全体

```
1. 環境チェック（CMUX_SOCKET_PATH）
2. レビュー種別判定（code / spec）
3. 基準モード判定（デフォルト / 基準なし / 基準のみ）
4. レビュアーエージェント選択（デフォルト: 親と異なるエージェント）
5. レビュー素材の収集（diff または spec ファイル）
6. ワークスペース作成 + レビュアー起動
7. プロンプト検出（ポーリング）
8. レビュープロンプト送信
9. 完了検出（段階的ポーリング）
10. 結果回収（read-screen --scrollback）
11. 結果を構造化レポートに整形
12. 親セッションでユーザーに報告
13. ワークスペースクリーンアップ
```

## 使用例

### コードレビュー（デフォルト）

```
ユーザー: 「この diff をセカンドオピニオンして」
ユーザー: "Get a second opinion on this diff"
エージェント: diff 取得 → Codex 起動 → レビュープロンプト送信 → 結果回収 → 報告
```

### 仕様書レビュー

```
ユーザー: 「auth-feature の仕様書のセカンドオピニオンをもらって」
エージェント: .specs/auth-feature/ 読み取り → Codex 起動 → 仕様書レビュー → 報告
```

### 自由レビュー（基準なし）

```
ユーザー: 「自由にレビューしてもらって」
エージェント: review_rules.md を渡さない → レビュアーが独自の判断でレビュー
```

### 厳密レビュー（基準のみ）

```
ユーザー: 「ルールに従って厳密にレビューして」
エージェント: review_rules.md のみ渡し、「追加の視点」指示なし
```

### レビュアー指定

```
ユーザー: 「Claude Code にレビューさせて」
エージェント: デフォルトの Codex ではなく Claude Code をレビュアーとして起動
```

## 基準モードの解説

### デフォルトモード（推奨）

review_rules.md をレビュアーに渡し、さらに「追加の視点があれば」と依頼。一貫性（共通基準）とフレッシュな視点のバランスが取れる。

### 基準なしモード

review_rules.md を渡さない。レビュアーが自身の知識と判断でレビュー。完全に独立した視点が欲しい場合、または review_rules.md が存在しない場合に使用。

### 基準のみモード

review_rules.md を渡し、厳密に基準に従うよう指示。コンプライアンスチェックや、プライマリレビューがルール違反を見落としていないか確認したい場合に使用。

## なぜ inspection-report.md を渡さないのか？

仕様書レビューの場合、既存の spec-inspect レポートはレビュアーに渡しません。これにより、セカンドオピニオンが真に独立したものになります。レビュアーは spec-inspect が見落とした問題を発見したり、spec-inspect の懸念を検証したりできます。

2つのレポートの比較は、ユーザー（または親エージェント）の責任です。

## プロンプトのコツ

- 大きな diff の場合、プロンプトが切り詰められることがあります。diff が約2000行を超える場合は、最も重要なファイルに焦点を当ててください。
- 仕様書レビューでは、3つのファイル（requirement.md, design.md, tasks.md）全てがプロンプトに含まれます。
- レビュアーはプロンプト内のコンテンツを見ますが、ファイルシステムへのアクセスはありません。

## エラーケース

### レビュアーエージェントが未インストール

`codex`（または指定されたエージェント）が見つからない場合、代替を提案します:

```
「Codex がインストールされていません。Claude Code または Gemini CLI を代わりに使いますか？」
```

### diff が空

`git diff HEAD` が何も返さない場合、レビュー対象がありません。変更を加えるか、別の diff 範囲を指定してください。

### review_rules.md が見つからない

これはエラーではありません。自動的に「基準なし」モードにフォールバックします。review_rules.md を生成するには `spec-rules-init --with-review-rules` を実行してください。

## 関連スキル

- **cmux-delegate**: 汎用タスク委任（レビュー特化ではない）。エージェントにレビューではなく作業をさせたい場合に使用。
- **cmux-fork**: 現在の会話をフォーク。同じ会話を新しいペインで続けたい場合に使用。
- **spec-inspect**: 自動化された仕様書品質チェック（セカンドオピニオンとは異なる。spec-inspect は事前定義チェック、セカンドオピニオンは AI 判断）。
