# cmux-delegate リファレンスガイド

## 概要

cmux-delegate は別の cmux ペインまたはワークスペースに AI エージェントを起動し、タスクを送信して、完了を監視し、結果を親セッションに回収します。

## 委任フロー全体

```
1. 環境チェック（CMUX_SOCKET_PATH）
2. リクエスト解析（タスク、エージェント、方向、ディレクトリ）
3. サーフェス作成（new-workspace または new-split）
4. ディレクトリ移動（ディレクトリ横断の場合）
5. エージェント起動
6. プロンプト検出（ポーリング）
7. タスク送信
8. 完了監視（段階的ポーリング）
9. 結果回収（read-screen --scrollback）
10. ユーザーに報告
11. クリーンアップ（オプション）
```

## 使用する cmux コマンド

### サーフェス作成

```bash
cmux new-workspace                    # 新ワークスペース（delegate のデフォルト）
cmux new-split right                  # 右にペイン分割
cmux new-split down                   # 下にペイン分割
```

出力: `OK surface:{N} workspace:{N}`

### コマンド送信

```bash
cmux send --surface surface:{N} "text\n"       # テキスト + Enter 送信
cmux send-key --surface surface:{N} return      # Enter のみ送信
```

### 画面読み取り

```bash
cmux read-screen --surface surface:{N}                  # 現在の画面
cmux read-screen --surface surface:{N} --scrollback 500  # 履歴付き
```

### クリーンアップ

```bash
cmux close-workspace --workspace workspace:{N}   # ワークスペース終了
cmux close-surface --surface surface:{N}          # サーフェスのみ終了
```

## 使用例

### Claude Code に委任（デフォルト）

```
ユーザー: 「別ペインでテストを実行して結果を教えて」
エージェント: 新WS作成 → claude 起動 → タスク送信 → 監視 → 結果報告
```

### Codex に委任

```
ユーザー: 「Codex にこの diff をレビューしてもらって」
エージェント: 新WS作成 → codex 起動 → レビュータスク送信 → 意見回収
```

### ディレクトリ横断

```
ユーザー: 「gslide-generator でテストを走らせて」
エージェント: 新WS作成 → cd ~/Documents/zenchaine/gslide-generator → claude 起動 → "テスト実行" 送信
```

### 方向指定付き

```
ユーザー: 「このタスクを右ペインに委任して」
エージェント: 新WSではなく右にペイン分割して委任
```

## 段階的ポーリングパターン

ポーリング間隔は時間経過に応じて段階的に長くなります:

| フェーズ | 経過時間 | 間隔 | 理由 |
|---------|---------|------|------|
| 高速 | 0〜60秒 | 5秒 | ほとんどのタスクは短時間で完了 |
| 中速 | 60〜300秒 | 10秒 | 長いタスクには頻繁なチェック不要 |
| 低速 | 300秒〜 | 30秒 | 非常に長いタスク、過剰なポーリングを回避 |

ポーリング間、親エージェントは他の作業やユーザーへの進捗報告が可能です。

## プロンプト検出パターン

各エージェントには入力待ちを示す特徴的なプロンプトパターンがあります:

| エージェント | 典型的なプロンプト |
|-------------|-----------------|
| Claude Code | 行頭の `>` またはセッション情報後のカーソル |
| Codex | カーソル付きプロンプト、セッションバナー表示 |
| Gemini CLI | `>` またはプロンプトインジケータ |

親エージェントは画面出力を読み取り、エージェントの準備状態を判断します。正確なパターンマッチングは不要で、LLM による画面内容の解釈で十分です。

## エラー復旧

### エージェントが起動しない

1. エージェントコマンドの存在確認: `which claude`
2. サーフェスが作成されたか確認: cmux 出力のハンドルを確認
3. 画面を読み取ってエラーメッセージを確認

### タスク実行中のクラッシュ

1. `cmux read-screen` でエラー出力やシェルプロンプト（エージェントプロンプトでない）を検出
2. ユーザーにエラーを報告
3. 必要に応じてエージェントを再起動してリトライ

### タスクが停滞している場合

1. 画面を読み取って現在の状態を確認
2. ユーザーに進捗を報告
3. 待機するかキャンセルするかユーザーに判断を委ねる

## 関連スキル

- **cmux-fork**: 現在の会話をフォーク（コンテキスト保持）。現在のセッションのコピーが欲しい場合に使用。
- **cmux-second-opinion**: 別の AI からレビューや意見を得るための特化型委任。
