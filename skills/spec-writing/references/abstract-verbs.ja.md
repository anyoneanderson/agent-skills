# 抽象動詞の語彙

この表は、日本語の抽象的な処理説明を検査するための一次ソースである。`Pattern` との一致は検査候補を選ぶために使い、一致した事実だけでは WARNING にしない。同じ文、箇条書き、段落、またはシーケンス図の近接するメッセージに `Required evidence` があれば報告しない。

| ID | Pattern | Ambiguity | Required evidence | Bad example | Rewritten example |
|---|---|---|---|---|---|
| AV-001 | `写像する` | メモリ上の変換、永続化、イベント送信のどれを実行するか判断できない。 | 主体、開始条件または入力、変換、保存、送信のうち実際に行う動作、その結果を渡す実在するすべての渡し先を書く。数学的な写像では、入力集合、変換規則、出力集合を定義してもよい。 | AI SDK の step を `Run.steps` と `agent-step` イベントへ写像する。 | AI SDK が step の開始を通知したら、orchestrator は進捗を `Run.steps` へ保存し、同じ進捗を `agent-step` イベントとしてクライアントへ送信する。 |
| AV-002 | `束縛する` | 値を関連づけるのか、ハッシュへ含めるのか、検証または認可に使うのか判断できない。 | 主体、入力または開始条件、関連づけ、計算のうち実際に行う動作と、フィールド、レコード、メッセージ、受信者のうち結果を実際に渡す先を書く。 | tenant と actor を `taskRequestDigest` へ束縛する。 | orchestrator がタスクリクエストを受信したら、tenant と actor をダイジェストの入力へ含めて値を計算し、`taskRequestDigest` をリクエストとともに実行キューへ送信する。 |
| AV-003 | `安全側へ倒す` | どの失敗に対して、何を拒否または停止し、結果や根拠をどこへ渡すか判断できない。 | 主体、失敗条件、拒否、停止、代替処理のうち実際に行う動作と、呼び出し元、ログ、状態、レスポンスのうち結果を実際に渡す先を書く。 | ポリシーの参照に失敗したら安全側へ倒す。 | ポリシーサービスがタイムアウトした場合、認可ゲートウェイはリクエストを拒否し、呼び出し元へ再試行可能なエラーを返して、理由を監査ログへ保存する。 |
| AV-004 | `収束させる` | 比較する値、目標状態、進捗の保存先、停止条件を判断できない。 | 主体、反復の開始条件、比較または更新の動作、目標状態、結果の渡し先、停止条件を書く。 | レビューの指摘を収束させる。 | 各レビュアーの応答後に、orchestrator は未解決の指摘 ID と採用した修正を比較し、残った ID を `review-state.json` へ保存して、未解決の ID がなくなるか3回の処理を終えた時点で停止する。 |

## 検査時の注意

- Pattern が現れただけでは WARNING にしない。
- 近接する文章またはシーケンス図のメッセージから一意に追える情報は、同じ処理の証拠として扱う。
- 通常のコードブロック、ログ、設定例、識別子だけに Pattern がある場合は検査対象から外す。
- 数学上の用法は、入力、変換規則、出力が定義されていれば合格とする。
- 修正文に仕様から確定できない動作が必要な場合は、値を作らず不足項目を名前つきのプレースホルダーで示す。
