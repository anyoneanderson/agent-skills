# 投影整合の検査

仕様書一式を読み込んだ後に、この検査を実行する。
生成契約は[spec-generatorの投影整合パス](../../spec-generator/references/projection-consistency.ja.md)
を参照し、検査の詳細はこのファイルに置く。
workflow plannerへ同じ規則を複製しない。

## 検査項目

| ID | 検査内容 |
|----|----------|
| PI-001 | `requirement.md`で定義したRequirement IDと、`design.md`、`tasks.md`、`test.md`のすべての参照を比較する |
| PI-002 | §0、§12、見出しと、それらが要約する現在の本文の設計判断を比較する。§12は存在する場合の末尾の要約または結論セクションを指す |
| PI-003 | §1トレーサビリティ表と、現在のRequirement IDおよび設計カバレッジを比較する |
| PI-004 | 同じ設計判断を表す箇所の数値基準と方式名を比較する |
| PI-005 | 変更した要求と設計判断が`tasks.md`と`test.md`へ投影されているか、対象外の理由が明記されていることを確認する |

構造化されたIDと値は完全一致で比較する。
要約文は意味で比較し、文字列の完全一致だけで同じ主張だと判定しない。
同じ数値や名称でも、周囲の文脈から異なる事実を表す場合は比較対象にしない。

## finding

- 未定義のRequirement ID参照は、既存のCheck 1で`CRITICAL`として報告する。
- 古い要約、トレーサビリティ行、数値、方式名、タスク、テストへの投影は`WARNING`として
  報告し、オーケストレーションされたinspect gateから仕様生成へ戻せるようにする。
- findingには、変更元の事実、古い投影先のファイルと行、投影先が表すべき現在の値または
  関係を含める。
- 残った古い値が意図した履歴または例示であり、その役割が明記されている場合はfindingを
  出さない。

変更した事実の影響を受ける投影先をすべて確認してから検査を終える。
古い投影を1件見つけても、残りの投影先の検査を省略しない。
